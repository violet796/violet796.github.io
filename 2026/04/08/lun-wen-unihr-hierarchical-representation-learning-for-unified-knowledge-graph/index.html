论文背景

真实世界的知识图谱被广泛应用在各种领域，如问答和自然语言处理等。传统三元组不能充分表示真实世界，因此，某些包含超关系、时序知识和嵌套事实等额外信息的知识图谱被用于建模事件。

  • 超关系知识图谱：除了普通三元组以外，在关系上还包含一些额外信息，如:（(奥本海默，受教育于，哈佛大学)，学位：学士，专业：化学）
  • 时序知识图谱：除了普通三元组外，还包含时序信息，如：(勒布朗詹姆斯，效力于，迈阿密热火，[2011-2014])
  • 嵌套事实知识图谱：除了普通三元组外，还包含三元组之间的关系，如：((奥本海默，出生于，纽约)，蕴含，(奥本海默，国籍为，美国))

三类知识图谱示意如下图所示:

现有研究主要有以下两个问题:

1）这些知识图谱不能反应有多种异构事实的真实场景，只能把它们分离，单个建模。

2）先前基于三元组知识图谱嵌入的方法证明了分层事实语义建模的有效性，但是超越三元组的表示形式较为复杂，这些方法无法实现全面的层级化语义建模，无法泛化到其它事实类型。

因此，提出一种针对具有多关系的真实世界知识图谱的统一的层次表示学习方法是值得研究的。

本文提出UniHR，一种统一层次表示学习框架，由层次数据表示模块(HiDR)和层次结构学习模块(HiSL)组成。

  • HiDR负责将不同知识图谱在不丢失信息的情况下转换成统一的三元组结构
  • HiSL通过事实内消息传递捕获局部语义信息，再利用事实间消息传递丰富全局结构信息以获得更好的节点嵌入

更新的节点嵌入将会被送入解码器用于链路预测。

论文的贡献可以总结为以下三点：

  • 强调了不同知识图谱统一表示，统一学习的重要性
  • 提出了第一个统一的层次知识图谱表示学习框架UniHR
  • 方法在不同数据集展示了良好的效果

相关工作

  • 超关系知识图谱：早期的超关系知识图谱补全只针对局部信息或全局信息。方法要么只关注了事实内交互(一个事实内部的元素)、要么只关注了事实间交互（共享了实体的事实）。最新的方法尝试对该知识图谱进行多层建模，但是针对该知识图谱的分层方法很难泛化到其它类型的事实中。
  • 嵌套知识图谱：23年首次被提出，但是现有的方法由于嵌套知识图谱表示的复杂性，很难掌握事实内的语义信息。
  • 时序知识图谱：针对该类型图谱的方法聚焦于设计杰出的时间感知模块，如用四阶张量分解+时间正则、几何空间建模单个事实内的方法，和沿着时间轴得到实体领域子图的的建模事实间的方法。没有一个方法可以同时捕捉两个层面。

论文方法

UniHR总体架构如下图所示：

分为以下三步:

  • 1）给定任意类型的知识图谱，记为\(G\)，通过HiDR将其转换成\(G^{HiDR}\)

  • 2）\(G^{HIDR}\)将会被HiSL模块编码，以增加特定事实的语义信息和，和整个图中不同事实的结构信息

  • 3）解码阶段，更新后的节点和边嵌入向量被序列化后输入Transformer模块以优化模型性能

Hierarchical Data Representation（HiDR）模块

把实体和关系全都划分成三个类型。

节点的三个类型包含:

  • 原子节点：本身存在于知识图谱中的实体节点，记作\(V_a\)
  • 关系节点：将关系抽象成节点，记作\(V_r\)
  • 事实节点：将三元组事实抽象成节点，记作\(V_f\)

关系的三个类型包含：

  • 原子关系：本身存在于知识图谱中的关系和HKG中的value，记作\(R_a\)
  • 连接关系：HiDR定义的专属关系，对应关系{has relation,has head entity,has tail entity}，记作\(R_c\)
  • 嵌套关系：两个事实之间的关系，记作\(R_n\)

由此衍生出来的三种事实类型包含:

  • 原子事实：本就存在于知识图谱中的事实，记作\(F_a\)
  • 连接事实：对应三元组(f,has relation,r)，(f,has head entity,h)，(f,has tail entity,t)，记作\(F_c\)
  • 嵌套事实：两个原子事实\(\alpha,\beta\)之间存在某种关系\(r\)，\((\alpha,r,\beta)\)即为嵌套事实，记作\(F_n\)

定义\(G^{HiDR}=\{V^{HiDR},R^{HiDR},F^{HiDR}\}\)

其中，\(V^{HiDR}=V_a \cup V_r \cup V_f,R^{HiDR}=R_a \cup R_c \cup R_n,F^{HiDR}=F_a \cup F_c \cup F_n\)。

①对于超关系知识图谱:

\(F_c\)除了上述连接事实以外，还包含由事实，及事实上的键值对构成的三元组。具体为(f,k,v)。\(F_n=\varnothing\)，因为该类型知识图谱事实之间没有嵌套关系。

②对于嵌套知识图谱：

\(F_c,F_a\)同定义，\(Fn=\{(f_1,R,f_2)|f_i\in F_{NKG}\}\)。

③对于时序知识图谱：

将时序知识图谱转换成特殊的超关系知识图谱，对于每个四元组\((h,r,t,[t_1,t_2])\)，先将其转换为\((h,r,t,begin:t_1,end:t_2)\)的类HKG形式，再按照HKG的模式转换成\(G^{HiDR}\)。

Hierarchical Structure Learning (HiSL)模块

HiDR定义了大量的额外节点，为了避免引入大量的训练参数，论文设计一个层级结构学习模块（HiSL），分成三个步骤实现:

表示初始化

对于HKG，事实上的键值对k:v，key作为连接关系,value作为原子关系

初始化所有原子实体的嵌入\(H_a\in R^{|v_a| ×d}\)以及所有关系（包含连接关系、嵌套关系）的嵌入\(E\in R^{|R^{HiDR}| ×d}\)，再初始化原子关系节点嵌入\(H_r\in R^{|v_r| ×d}\)，该嵌入由原子关系嵌入\(E_a\)经过可学习的投影矩阵\(W_r\)变化而来，其中\(E_a\in E\)，具体表达式为\(H_r=E_a·W_r\)。

初始化事实节点嵌入\(H_f\)以学习事实内的关键信息，具体公式为: \[ h_f=f_m([h_h;h_r;h_t]) \] \(h_h,h_r,h_t\)是一个事实内头尾实体和关系的嵌入，通过连接操作变成一个维度为\(3d\)的向量。\(f_m\)是一个一层的多层感知机，将该向量维度转换为\(d\)。

对于时序知识图谱的时间节点t，使用Time2Vec将其编码，具体公式为： \[ h_{\tau}=w_psin(f_p(\tau))+f_{np}(\tau) \] \(f_p,f_{np}\)均是一个一层的多层感知机，分别作为周期函数和非周期函数，将一个时间标量转换为\(d\)维向量。\(w_P\)是一个一维的可学习参数，用于缩放周期特征。

事实内消息传递

针对每一个事实，构建一个子图，子图只包含事实节点、在该事实内与事实相连的原子节点、连接的边，旨在学习完整的事实成立条件。

给定初始嵌入，逐层次学习，在学习的过程中，通过注意力机制不断更新事实节点和原子节点：

  • 事实节点的更新可以理解成：把各个原子节点打包，聚合所有组成元素的信息，更新完整的事实语义
  • 原子节点的更新可以理解成：让各个原子节点获得在该事实下的表示

注意力权重的计算公式为： \[ \alpha_{i,j}^l=\frac{exp(W^l(\sigma(W^l_{in}h^l_i+W^l_{out}h_j^l)))}{\sum\limits_{j'\in N_i}exp(W^l(\sigma(W^l_{in}h_i^l+W^l_{out}h^l_{j'})))} \] \(h_i^l,h_j^l\)为中心实体\(i\)和邻居实体\(j\)在第\(l\)层的嵌入。\(W_{in}^l\)为第\(l\)层的可学习矩阵，类似于Transfomer中将嵌入转换为Q向量的矩阵，\(W_{out}^l\)的作用则是将嵌入转换为K，V向量的聚合。\(W^l\)则是第\(l\)层的一个评分矩阵，用于学习组合的合理性。

个人理解：尽管论文当中子图的所有结点均能当作中心结点，但是对于非事实结点而言，其邻居节点只有事实结点一个结点，故注意力权重得分只有一项且为1，更新的时候也是只考虑该事实带来的语义

注意力得分的计算过程可以理解成：

从中心实体出发，通过两个可学习矩阵，进行向量的变换，得到一个类似于我的身份是xx，我的邻居能给我提供的信息是xx，它的信息是xx的向量，通过\(W^l\)给出评价，再经过\(Softmax\)进行归一化，得到最后的注意力分数。

得到注意力权重后，更新结点表示，公式如下： \[ h_i^l=h_i^l+\sum_{i\in N_i}\alpha^l_{i,j}·W^l_{out}h^l_j \]

事实间消息传递

目的是学习全局结构，丰富结点语义信息，消息传递将会在整张图\(G^{HIDR}\)上执行。

为了不增加要训练的参数，并更好的融合关系和实体信息，使用非参数循环算子\(\phi(·)\)，它可以将两个\(d\)维的向量转换为一个\(d\)维向量，公式如下: \[ \phi(h_j,e_r)=h_j\star e_r=F^{-1}((Fh_j)\odot(\overline{Fe_r})) \] \(F,F^{-1}\)分别代表傅里叶变化矩阵及其逆矩阵。通过该操作，可以把关系+实体变成连贯的语义。

为了更好的掌握图的异质性，将边从两个维度进行分类，并分别设置可学习参数：

  • 方向维度：包含两个方向\(\lambda(r)\in\{forward,reverse\}\)。GNN消息传递的方向是从邻居节点到中心节点，如果图谱中关系方向和消息传递方向不一致，即，图谱中的关系方向是中心节点到消息节点，那么该关系对应的方向为\(reverse\)

  • 类型维度：包含三种类型\(\tau(r) \in\{connected\;relation,atomic\;relation,nested\;relation\}\)，分别对应了关系的三种类型。

对于方向维度，设置两个可学习参数\(W_{\lambda(r)}\in R^{d×d}\)，分别代表正向和逆向，用于学习不同方向关系的贡献；对于类型维度，设置三个可学习参数\(w_t(r)\in R^1\)，用于学习三种类型关系的贡献。

通过下式更新实体表示: \[ h_i^{l+1}=\sum_{(r,j)\in N(i)}\sigma(w_{\tau(r)}^l)W^l_{\lambda(r)}\phi(h_j^l,e^l_r)+W^l_{self}h^l_i \] \(N(i)\)是所有与\(i\)相连的节点，\((r,j)\)是对应的关系及节点，\(\sigma\)是\(sigmoid\)函数。通过聚合周围所有实体及关系的信息，再加上自环项保留自身语义信息，实现对节点嵌入的更新。

对于关系表示，每一层也会按照下式更新: \[ e^{l+1}_r=W^l_{rel}e^l_r \] \(W_{rel},W_{self}\)学习的是自己的信息应该保留多少、自己应该怎么变。

链路预测解码器

采用TransFomer的解码器，将更新好的嵌入，用\([M]\)掩盖需要预测的部分，获得最后一层输出的\([M]\)向量，记作\(h_{pre}\)，并计算所有候选答案的概率分布。 \[ P=Softmax(f(h_{pre}[E;H])^T)\in R^{|R|+|V|} \] \(P\)代表所有候选实体的分数，\(f()\)是一个一层感知机，用于映射预测向量，\([E;H]\)是所有候选的边和实体的矩阵。

损失函数使用交叉熵函数，具体如下： \[ L=\sum_{t=0}^{|R|+|V|}y_tlogP_t \] \(y_t,p_t\)分别代表第\(t\)类的真实标签和概率。

代码结构

代码由数据集和五个主要文件组成：

NKG数据集：DBHE、FBH、FBHE，包含原子三元组，词汇表由两个特殊编码[PAD]、[MASK]，关系，实体和原子事实构成

HKG数据集：wikipeople、wd50k，词汇表由两个特殊编码[PAD]、[MASK]、关系、实体构成，key作为关系，value作为实体

TKG数据集：wikidata12k，词汇表由两个特殊编码[PAD]、[MASK]、实体和时间戳构成

五个主要文件:

  • run.py：训练/测试入口
  • reader.py：数据读取、样本构造、batch组织
  • graph.py：把训练事实构造成全局图(HIDR/HiSL用)
  • transformer.py：模型主体：全局图编码 + 局部Transformer解码
  • evaluation.py：filtered evaluation 和指标计算

def prepare_batch_data(inst, vocabulary: Vocabulary, max_arity, max_seq_length):
    # inst: [input_ids, input_mask, mask_position, mask_label, query_type]
    #输入的token 索引
    input_ids = np.array(inst[0]).astype("int64")
    #有效的掩码位置
    input_mask = np.array(inst[1]).astype("int64")
    #当前掩码位置
    mask_position = np.array(inst[2]).astype("int64")
    #当前掩码token的索引
    mask_label = np.array(inst[3]).astype("int64")
    #需要查找到token类型
    query_type = np.array(inst[4]).astype("int64")
    input_mask = np.outer(input_mask, input_mask).astype("bool")

方法先获取相关信息，再利用outer()方法，判断哪些token直接可以进行注意力，将input_mask变成一个n*n的矩阵。

    norm = in_deg.__pow__(power)
    norm[np.isinf(norm)] = 0
    return torch.from_numpy(norm)

def node_norm_to_edge_norm(graph, node_norm):

    graph.apply_edges(lambda edges: {'norm': edges.dst['norm'] * edges.src['norm']})
    norm = graph.edata.pop('norm').squeeze()
    return norm

class MultiHeadAttention(nn.Module):
    def __init__(self, hidden_dim: int, heads: int, dropout_prob: float, use_edge: bool, remove_mask: bool, bias: bool, use_node: bool) -> None:
        super().__init__()
        #判断维度是否能除尽头数
        assert hidden_dim % heads == 0
        #每一头的维度
        self.dim = hidden_dim // heads
        #头数
        self.heads = heads
        #获取q,k,v矩阵
        self.query = PrepareForMultiHeadAttention(hidden_dim, heads, bias, use_node)
        self.key = PrepareForMultiHeadAttention(hidden_dim, heads, bias, use_node)
        self.value = PrepareForMultiHeadAttention(hidden_dim, heads, True, use_node)
        self.pos = PrepareForMultiHeadAttention(hidden_dim, heads, True, use_node)
        #softmax层
        self.softmax = nn.Softmax(dim=-1)
        self.output = nn.Linear(hidden_dim, hidden_dim)
        self.dropout = nn.Dropout(p=dropout_prob)
        #消融实验
        self.use_edge = use_edge
        self.remove_mask = remove_mask
        self.scale = 1 / math.sqrt(self.dim)
        # trasformer-xl
        self.r_w_bias = nn.Parameter(torch.Tensor(heads, self.dim)) # u
        self.r_r_bias = nn.Parameter(torch.Tensor(heads, self.dim)) # v

class PrepareForMultiHeadAttention(nn.Module):
    def __init__(self, hidden_dim: int, heads: int, bias: bool, use_node: bool) -> None:
        super().__init__()
        self.heads = heads
        self.use_node = use_node       

        if self.use_node is True:
            #可训练参数为hidden_dim * hidden_dim + hidden_dim个，目的是学习每个token的映射
            #如映射到k空间 q空间 vko
            self.layer_s=nn.Linear(hidden_dim,hidden_dim)
            self.layer_r=nn.Linear(hidden_dim,hidden_dim)
            self.layer_o=nn.Linear(hidden_dim,hidden_dim)
            self.layer_a=nn.Linear(hidden_dim,hidden_dim)
            self.layer_v=nn.Linear(hidden_dim,hidden_dim)
        else:
            self.linear = nn.Linear(hidden_dim, hidden_dim, bias=bias)

PrepareForMultiHeadAttention()的前向传播流程如下:

def forward(self, x : torch.Tensor):
    #shape为 batch*seq_len*hidden_dim
    shape = x.shape[:-1]

    if self.use_node is False:
        x = self.linear(x)
    else:
        device=x.device
        #0代表batch_size 1代表一条三元组被扩展后的维度 2代表hiddendim
        max_seq_len=x.size(1)
        #主体的掩码
        mask_s = torch.tensor([1]+[0]*(max_seq_len-1)).to(device)
        #关系的掩码
        mask_r = torch.tensor([0,1]+[0]*(max_seq_len-2)).to(device)
        #客体的掩码
        mask_o = torch.tensor([0,0,1]+[0]*(max_seq_len-3)).to(device)
        #key的掩码
        mask_a = torch.tensor([0,0,0]+[1,0]*int(((max_seq_len-3)/2))).to(device)
        #value的掩码
        mask_v = torch.tensor([0,0,0]+[0,1]*int(((max_seq_len-3)/2))).to(device)

        #mask_s[:,none]把实体的掩码变成二维的
        x_s=self.layer_s(torch.mul(x,mask_s[:,None].expand(-1,x.size(-1))))
        x_r=self.layer_r(torch.mul(x,mask_r[:,None].expand(-1,x.size(-1))))
        x_o=self.layer_o(torch.mul(x,mask_o[:,None].expand(-1,x.size(-1))))
        x_a=self.layer_a(torch.mul(x,mask_a[:,None].expand(-1,x.size(-1))))
        x_v=self.layer_v(torch.mul(x,mask_v[:,None].expand(-1,x.size(-1))))

        x=(x_s+x_r+x_o+x_a+x_v) 

        return x.reshape(*shape, self.heads, -1)

如果use_node=False，代表不区分节点类型，所有节点共用一个线性层；如果区分节点类型，则对每个不同类型的实体采用不同的线性层。mask_s[:,none]把实体的掩码变成二维的，从1× max_seq_len变成max_seq_len×1，刚好对应每个token的位置，再利用expand将其扩展为max_seq_len×hidden_dim维度，与原token嵌入逐元素相乘，最后只留下头实体token，其余位置均为0。

对于其它操作同理。不同类型的实体分别进入不同的线性层，将结果相加，便是最后的x。

最后，将x变换成多头注意力所需要的形状，与多头注意力的头数有关。例如，最开始的输入为[24,7,256]，头数为4，那么最后的输出将会变成[24,7,4,64]，这代表在最开始：

  • batch_size = 24，即有24条数据
  • max_seq_len=7，代表每一个三元组都会被扩展为7个token
  • hidden_dim=256，代表每个token维度为256

但是最后需要把一个256维度的token分为4个64维度的token用于多头注意力机制。

torch.einsum()在做点积，输入的query和key的维度均为[batch_size,max_seq_len,head,dim]，做点积后输出的维度应该为[batch_size,head,max_seq_len,max_seq_len]，输出的结果为不同token之间的注意力分数。类似于： \[ Scores = Q·K^T \] 如果remove_mask = True，那么不同键值对间的关系将置为False，代表不同键值对之间将互不可见。

三个分支代表含义为:

  • 如果\(i=3\)，说明其为第一个键值对，那么它将无法看到所有其之后的键值对
  • 如果\(i=seq\_len-2\)，说明其为最后一个键值对，那么它将无法看到所有其之前的键值对
  • 其余情况则设置为，既不能看见之前的键值对，也不能看见之后的键值对

scores将不可见的位置设置为-100000，对最后一维经过softmax后，不可见位置将几乎不会影响attention分数。

scores维度为[batch_size,head,max_seq_len,max_seq_len]，对最后一维进行softmax，指的就是在某条数据的某个注意力块中，每个token和其它token的注意力权重。

得到权重后，再做一次内积，每个token得到的就是带有对其它token注意力的新的嵌入。

        self.rel_embedding = nn.parameter.Parameter(torch.Tensor(rel_num*2+6, dim))
        nn.init.normal_(self.rel_embedding, mean=0, std=0.02)
    else:
        #如果是TKG，将时间一并加入，不更新嵌入，学习整体的结构
        self.ent_embedding = nn.parameter.Parameter(torch.Tensor(ent_num, dim),requires_grad=False)
        nn.init.normal_(self.ent_embedding, mean=0, std=0.02)
        self.rel_embedding = nn.parameter.Parameter(torch.Tensor(rel_num*2+6, dim))
        nn.init.normal_(self.rel_embedding, mean=0, std=0.02)

Transformer()的前向传播流程如下:

def forward(self, input_ids, input_mask, mask_position, mask_output, edge_labels):
    embedding = self.globl().to(input_ids.device)
    x = torch.nn.functional.embedding(input_ids, embedding)
    x = self.input_dropout(self.input_norm(x))
    #不同token之间的关系变成了一个向量
    edge_query = self.edge_query_embedding(edge_labels)
    edge_key = self.edge_key_embedding(edge_labels)
    edge_value = self.edge_value_embedding(edge_labels)

    #解码过程
    for layer in self.layers:
        x = layer(x, input_mask, edge_key, edge_value, edge_query)
    #此时每一个token都包含了对应的上下文，取出掩码位置
    x = x[torch.arange(x.shape[0]), mask_position]
    x = self.output_linear(x)  # x(batch_size, hiddem_dim)
    x = self.output_act(x)
    x = self.output_norm(x)
    y = torch.mm(x, embedding.transpose(0, 1)) + self.output_bias
    y = y.masked_fill(mask_output == 0, -100000)
    return y
